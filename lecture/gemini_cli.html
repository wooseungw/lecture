Gemini CLI - 설치 및 활용 가이드

1. Gemini CLI란?

Gemini CLI는 텍스트 기반의 터미널 환경에서 Google의 강력한 AI 모델 Gemini를 직접 사용할 수 있는 커맨드 라인 도구입니다.
브라우저 없이, 가벼운 터미널 창에서 바로 AI와 대화하고 파일 생성, 코드 작성, 웹 검색 등 다양한 작업을 수행할 수 있습니다.



  • 무료 사용: 신용카드 등록 없이 하루 1,000회 요청 가능
  • 가볍고 빠름: 별도의 프로그램 설치 없이 터미널에서 즉시 실행
  • 강력한 기능: 단순 대화를 넘어 파일 시스템 제어, 웹 접근 등 확장 기능 제공
# Gemini CLI는 이런 대화형 인터페이스를 제공합니다.

Gemini > "풍선을 든 ASCII 고양이 그려 줘."

2. 사전 준비: Node.js 설치

Gemini CLI는 Node.js의 패키지 매니저인 `npm`을 통해 설치됩니다. 따라서 Node.js를 먼저 컴퓨터에 설치해야 합니다.

  • 웹 브라우저에서 'Node.js LTS 다운로드'를 검색하여 공식 사이트로 이동합니다.
  • LTS (Long Term Support) 버전 설치 파일을 다운로드합니다.
  • 설치 프로그램을 실행하고, 별다른 설정 변경 없이 '다음' 버튼을 눌러 설치를 완료합니다.

    • 설치 절차

    1. Node.js의 웹사이트에서 사용자의 OS를 조회해 설치 프로그램 방식을 자동을 보여줍니다.
    2. LTS 버전은 안정성과 호환성이 뛰어나므로, 최신 버전이 아닌 LTS 버전을 선택하는 것이 좋습니다.
    3. CLI를 통해 설치할 수 있지만, 처음 사용하는 경우에는 GUI 설치가 더 간편합니다.

    3. '작전실' 열기: 터미널 실행

    모든 명령어는 터미널(커맨드 라인 인터페이스)을 통해 입력됩니다. 운영체제에 맞는 터미널을 실행해야 합니다.

    운영체제별 터미널 실행법

    • Windows: `⊞ 시작` 메뉴에서 'PowerShell'을 검색한 후, 마우스 오른쪽 버튼을 클릭하여 '관리자 권한으로 실행'을 선택합니다.
    • macOS: `Spotlight` (⌘ + Space)를 열고 'Terminal'을 검색하여 실행합니다.
    # Windows PowerShell (관리자)

PS C:\WINDOWS\system32> _


# ---------------------------------
# macOS Terminal

user@MacBook-Air ~ % _

    4. Gemini CLI 설치하기

    앞서 실행한 터미널에 `npm` 명령어를 입력하여 Gemini CLI를 설치합니다. 설치는 보통 1분 이내에 완료됩니다.

    `-g` 옵션의 의미

    `-g` (global) 옵션은 Gemini CLI를 시스템 전역에 설치하여, 어떤 폴더에서든 `gemini` 명령어를 바로 사용할 수 있게 해줍니다.

    # 터미널에 아래 명령어를 그대로 입력하고 Enter를 누릅니다.

npm install -g @google/gemini-cli

# ---------------------------------
# macOS에서 권한 오류(EACCES)가 발생할 경우,
# 명령어 앞에 'sudo'를 붙여 다시 시도합니다.

sudo npm install -g @google/gemini-cli

    5. 첫 실행 및 Google 인증

    설치가 완료되면 `gemini` 명령어를 입력하여 프로그램을 실행합니다. 처음 실행 시, Google 계정으로 로그인하는 인증 절차를 거치게 됩니다.

    인증 과정

    터미널에 표시되는 URL을 웹 브라우저에 붙여넣고, 사용하려는 Google 계정으로 로그인하여 권한을 허용하면 인증이 완료됩니다.

    # 터미널에 'gemini'를 입력하여 실행합니다.

gemini

    7. 파일 및 코드 생성

    Gemini CLI는 단순한 텍스트 응답을 넘어, 요청에 따라 실제 파일을 생성하고 코드를 작성해주는 강력한 기능을 갖추고 있습니다.

    어떻게 동작하나요?

    AI가 파일 생성이나 수정이 필요하다고 판단하면, 사용자에게 변경 사항을 미리 보여주고 실행 여부를 묻습니다. 사용자가 동의하면 현재 디렉토리에 파일이 생성됩니다.

    8. 실시간 웹 정보 요약

    Gemini CLI는 내장된 웹 브라우징 기능을 통해 특정 URL의 콘텐츠를 가져와 요약하거나, Google 검색을 수행하여 최신 정보를 바탕으로 답변할 수 있습니다.

    활용 예시

    기술 문서 요약, 뉴스 기사 분석, 특정 주제에 대한 블로그 포스트 검색 등 정보 수집 및 분석 작업을 자동화할 수 있습니다.

    # 특정 URL의 내용을 가져와 요약 요청

Gemini › "web.fetch https://news.ycombinator.com 해서 상위 3개 기사 요약해 줘."

    9. VS Code 터미널에서 사용

    Visual Studio Code (VS Code)를 사용한다면, 별도의 경로 설정 없이 내 작업 환경에서 바로 Gemini CLI를 활용할 수 있습니다. VS Code의 통합 터미널은 현재 작업 중인 프로젝트 폴더를 기본 경로로 사용하기 때문입니다.

    사용 방법

    1. VS Code를 실행하고 프로젝트 폴더를 엽니다.
    2. 상단 메뉴에서 터미널(Terminal) > 새 터미널(New Terminal)을 선택하여 통합 터미널을 엽니다.
    3. 터미널이 열리면 자동으로 현재 프로젝트 폴더 경로로 설정됩니다.
    4. 이제 `gemini`를 입력하여 Gemini CLI를 바로 사용할 수 있습니다.
    # VS Code 통합 터미널 예시(Mac)

user@MacBook-Air ~/my_project % gemini
Welcome to Gemini!

    10. 문제 해결 (Troubleshooting)

    설치나 실행 과정에서 문제가 발생했을 때 참고할 수 있는 일반적인 해결 방법입니다.

    Windows 흔한 문제

    • "node 인식 안 됨": Node.js 설치 후, 새 PowerShell 창을 열어 다시 시도하세요.
    • EPERM 권한 오류: PowerShell을 '관리자 권한으로 실행'했는지 확인하세요.

    macOS 흔한 문제

    • EACCES 권한 에러: `npm install` 명령어 앞에 `sudo`를 붙여 실행하세요.
    # Windows 권한 오류 해결 예시
