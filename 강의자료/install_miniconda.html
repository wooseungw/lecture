Miniconda 설치 및 가상환경 생성

1단계: Miniconda 다운로드

Miniconda는 프로젝트별 독립된 개발 환경을 구축하는 데 필수적인 도구입니다.

Anaconda 공식 홈페이지에 접속하여 자신의 운영체제에 맞는 Miniconda Installers를 다운로드합니다.

Miniconda, 왜 사용하나요?

프로젝트마다 필요한 라이브러리 버전이 다를 때, Miniconda는 독립된 '가상환경'을 만들어 충돌을 막아줍니다. 이를 통해 여러 프로젝트를 안정적으로 관리할 수 있습니다.

2단계: 설치 및 사용자 범위 선택

다운로드한 설치 파일을 실행하고 라이선스에 동의합니다.

설치 유형을 선택하는 화면에서 'All Users (requires admin privileges)'를 선택합니다. 이 옵션은 PC의 모든 사용자가 Miniconda를 사용할 수 있게 합니다.

관리자 권한으로 실행

'All Users' 옵션을 선택하려면 설치 파일 자체를 '관리자 권한으로 실행'하는 것이 안정적입니다. (Windows 해당) 이는 시스템 폴더에 파일을 쓰는 데 필요한 권한을 부여합니다.

3단계: PATH 환경변수 설정

고급 설치 옵션 창에서 'Add Miniconda3 to the system PATH' 옵션을 체크합니다.

이 설정은 일반 터미널(cmd, PowerShell)에서도 conda 명령어를 바로 사용할 수 있게 해줍니다. 'Install' 버튼을 눌러 설치를 시작합니다.

PATH 설정 시 주의사항

이 옵션은 편리하지만, 시스템에 다른 Python 버전이 이미 설치된 경우 충돌을 일으킬 수 있습니다. Conda 사용이 처음이라면 체크하는 것이 편리하며, 문제 발생 시에는 재설치 시 이 옵션을 해제하고 'Anaconda Prompt'를 사용하는 것이 좋습니다.

4단계: Conda 환경 초기화

설치 후, 터미널에서 conda 명령어를 사용하기 위한 초기화 작업을 진행합니다.

Windows는 Anaconda Prompt, Mac은 터미널을 실행하고 아래 명령어를 입력하세요.

conda init

conda init은 무엇을 하나요?

사용자가 주로 사용하는 셸(bash, zsh, powershell 등)의 설정 파일에 conda를 활성화하는 스크립트를 자동으로 추가합니다. 이 작업을 통해 터미널이 시작될 때마다 conda를 바로 사용할 수 있게 됩니다.

5단계: 가상환경 생성

conda create 명령어를 사용하여 새로운 가상환경을 만듭니다. 이 환경은 다른 프로젝트와 완전히 분리된 공간이 됩니다.

conda create -n [가상환경이름] python=[파이썬버전]
# 예시: action 이라는 이름, python 3.12 버전
conda create -n action python=3.12

Proceed? [y]/n

생성 과정에서 설치될 패키지 목록을 보여주며 진행 여부를 묻습니다. y를 입력하고 Enter를 눌러 계속 진행합니다.

6단계: 가상환경 활성화

생성한 가상환경에 진입하여 작업을 시작합니다. conda activate 명령어를 사용합니다.

conda activate [가상환경이름]
# 예시: action 가상환경 활성화
conda activate action

활성화 확인 방법

명령어가 성공적으로 실행되면 터미널 프롬프트 맨 앞에 (base) 대신 (action)과 같이 현재 활성화된 가상환경의 이름이 표시됩니다.

프롬프트 앞에 (action)이 표시된 모습

7단계: 패키지 설치

가상환경이 활성화된 상태에서 프로젝트에 필요한 라이브러리(패키지)를 pip 또는 conda 명령어로 설치합니다.

# 예시: OpenCV, Jupyter Notebook 패키지 설치
pip install opencv-python
conda install jupyter
# 설치된 패키지 목록 확인
pip list

pip vs conda install

일반적인 Python 패키지는 pip로 설치하고, 데이터 과학 관련 패키지나 의존성 관리가 중요한 패키지는 conda로 설치하는 것을 권장합니다. 둘을 혼용하여 사용할 수 있습니다.

pip install 명령어로 패키지를 설치하는 모습

8단계: VSCode 인터프리터 설정

VSCode가 우리가 만든 가상환경을 사용하도록 설정합니다.
1. VSCode에서 Ctrl + Shift + P를 눌러 명령 팔레트를 엽니다.
2. Python: Select Interpreter를 검색하여 선택합니다.
3. 목록에서 방금 생성한 가상환경(예: 'action')을 선택합니다.

연동 확인

설정이 완료되면 VSCode 창의 오른쪽 하단에 선택한 가상환경의 이름과 Python 버전이 표시됩니다. 이제 해당 환경에서 코드를 실행하고 디버깅할 수 있습니다.
